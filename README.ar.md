# ServiceHub — وثائق عربية

> منصة متعددة المزودين لاكتشاف الخدمات وحجزها وإدارتها. تجمع رحلة العميل من البحث حتى التقييم، وتمنح مزود الخدمة والإدارة أدوات تشغيل واضحة.

**اللغات:** [English](README.md) · [Português (Brasil)](README.pt-BR.md)

> **الحالة: MVP مكتمل للـ portfolio — 16 يوليو 2026.** المسارات الأساسية موجودة، وآخر تحقق مستقل نجح فيه **220/220 اختبار API**. هذا لا يعني نشرًا عامًا أو جاهزية تشغيل مدفوعات حقيقية؛ الحدود أدناه مكتوبة بوضوح.

## الفكرة

بدل أن تُدار الحجوزات عبر رسائل ومكالمات وجداول منفصلة، يوحّد ServiceHub العملية:

```text
اكتشاف مزود → اختيار خدمة وموعد → حجز محمي → دفع → إشعار → تقييم
```

ويناسب المطاعم والصالونات والاستشارات والصيانة وكل خدمة تُحجز في وقت محدد.

## ما يقدمه النظام

| الدور | ما يستطيع فعله |
|---|---|
| **العميل** | البحث والفلاتر، صفحة مزود الخدمة، اختيار الموعد، حجز مؤقت أثناء الدفع، الإلغاء، التقييم، الرسائل، والإشعارات |
| **مزود الخدمة** | إدارة الخدمات، التوفر الأسبوعي والاستثناءات، الاطلاع على الحجوزات، وواجهات dashboard تشغيلية |
| **الإدارة** | اعتماد/تعليق المزودين، مؤشرات السوق، التقارير، النزاعات، الفئات، وخدمات التصدير والمدفوعات المستحقة |

## لماذا التصميم التقني مهم؟

- **لا حجزين متعارضين لنفس الموعد:** قاعدة PostgreSQL نفسها تفرض قيد `EXCLUDE USING gist` على الحجوزات النشطة.
- **الدفع قابل للتبديل:** Mock للتطوير والعرض، وStripe ضمن interface منفصل عند الإعداد الحقيقي.
- **صلاحيات وعزل بيانات:** أدوار Customer / Vendor / Admin مع فحوص ownership وIDOR في المسارات الحساسة.
- **قواعد تشغيل واضحة:** حجز مؤقت 5 دقائق، سياسة إلغاء، webhook idempotent، وتقييم واحد للحجز المؤهل.
- **هيكل قابل للصيانة:** Next.js للواجهة، NestJS للـ API، Prisma وPostgreSQL للبيانات، واختبارات Jest مستقلة.

## البنية

```text
المستخدمون
   ↓
Next.js Web App
   ↓
NestJS REST API
   ↓
Prisma → PostgreSQL
   ↓
Mock payments للتطوير / Stripe عند الإعداد الإنتاجي
```

| الطبقة | التقنية | دورها |
|---|---|---|
| الواجهة | Next.js 14 + TypeScript + Tailwind | رحلات العميل والمزود والإدارة |
| الـ API | NestJS 11 | Auth، الخدمات، التوفر، الحجوزات، الدفع، المراجعات، الرسائل، الإدارة |
| البيانات | PostgreSQL 16 + Prisma 5 | معاملات وmigrations وقيود النزاهة |
| الجودة | Jest + ts-jest | اختبارات الخدمة والأمان والتكامل |

## التشغيل محليًا

### API

```bash
cd apps/api
cp .env.example .env
npm install
npx prisma migrate deploy
npx prisma generate
npm run build
npm start
```

### Web

```bash
cd apps/web
npm install
npm run build
npm start
```

للتطوير: `npm run start:dev` داخل `apps/api` و`npm run dev` داخل `apps/web`.

## التحقق الأخير

تم في **16 يوليو 2026** تشغيل:

```bash

npx jest --runInBand
```

راجع [`docs/qa/acceptance-checklist.md`](docs/qa/acceptance-checklist.md) لمعرفة ما تم إثباته وما يزال يحتاج Browser QA أو تجربة تشغيلية.

## حدود العرض بشفافية

- `PAYMENTS_PROVIDER=mock` مخصص للتطوير والعرض؛ ليس دفعًا فعليًا.
- Stripe الحقيقي يحتاج secrets وwebhook وإعداد نشر واختبار E2E قبل أي استخدام تجاري.
- زر **Google (Demo)** محاكاة محلية؛ لا يتصل بـ Google ولا يمثل OAuth حقيقيًا.
- لا توجد مطالبة بأن المشروع منشور للعامة أو جاهز لإدارة مدفوعات حقيقية.
