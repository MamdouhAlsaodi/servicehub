ServiceHub

منصة موحّدة لاكتشاف الخدمات، الحجز، الدفع، وإدارة الأعمال.

عرض عربي تفصيلي يشرح النظام وصفحاته ومزاياه وخارطة التطوير.

العميل

اكتشاف، حجز، دفع، تقييم.

مزود الخدمة

خدمات، توفر، حجوزات، مؤشرات.

الإدارة

اعتمادات، تقارير، ونزاعات.

01 — المشكلة

الحجز المحلي غالباً موزع وغير قابل للقياس

  • العميل يتنقل بين رسائل واتساب ومكالمات ودفاتر مواعيد.
  • مزود الخدمة لا يرى مصدر بيانات واحداً للخدمات والتوفر والحجوزات.
  • الإدارة تحتاج إلى موافقات ومؤشرات تشغيل ونزاعات قابلة للمتابعة.
  • الحجز المتزامن قد ينتج عنه تعارضات مكلفة إن لم يُحمَ على مستوى قاعدة البيانات.
02 — القيمة

من الاكتشاف إلى التقييم في رحلة واحدة

اكتشف←اختر مزوداً←احجز موعداً←ادفع←قيّم التجربة

للعميل

قرار أسرع وثقة أعلى في الموعد والسعر والحالة.

للعمل

بيانات تشغيلية قابلة للقياس بدل متابعة يدوية مشتتة.

08 — أمان وموثوقية

لا يعتمد الحجز الآمن على الواجهة فقط

منع double booking

PostgreSQL EXCLUDE constraint يمنع نجاح حجزين متعارضين لنفس الموعد.

عزل الصلاحيات

Customer / Vendor / Admin مع tenant-scoped reads وفحوصات IDOR.

جلسات آمنة

JWT في HttpOnly cookies؛ لا token قابل للقراءة من JavaScript.

CSRF

double-submit token للطلبات المتغيرة في cookie-auth.

09 — قرار Demo واضح

الشفافية جزء من جودة المنتج

Google (Demo)

محاكاة محلية للعرض فقط. لا OAuth حقيقي، لا Google credentials، ولا هوية خارجية.

  • الدفع يعمل عبر Mock للتجربة؛ Stripe هو مسار الإنتاج عند تجهيز الأسرار وwebhook.
  • الإشعارات تستخدم REST polling كخيار MVP موثق، وليست WebSocket حالياً.
  • رابط التجربة محمي داخل Tailscale قبل مرحلة النشر العام.

11 — خطة الاختبار

بعد التجربة: evidence قبل أي إطلاق

1. تجربة يدوية

رحلات العميل وVendor وAdmin وتسجيل الملاحظات.

2. قبول API

صلاحيات، transitions، validation، وأخطاء متوقعة.

3. E2E

الحجز والدفع والإلغاء من المتصفح.

4. أداء وRace

تزامن، زمن استجابة، وقياس baseline.

12 — خارطة المستقبل

ما بعد تثبيت النسخة الأساسية

  • i18n كامل: Arabic RTL وEnglish LTR مع language switcher.
  • Lifecycle وتنبيهات: reminders 24h/1h، dedup، وإثبات booking race E2E.
  • تجاري: إعداد العمولة، CSV، طلبات payout، ومراجعة البلاغات.
  • مراسلة: رسائل مرتبطة بالحجز مع صلاحيات وpagination.
  • تشغيل: Swagger وCI وDocker وbackups وmetrics وrelease acceptance.
13 — سيناريو التجربة

كيف تجرّب النظام الآن

  1. افتح الرابط وسجل عبر Google (Demo) مع ملاحظة عبارة المحاكاة.
  2. اكتشف مزوداً وافتح الخدمات واختر موعداً.
  3. أنشئ حجزاً ثم جرّب Mock payment confirmation.
  4. راجع حجوزاتي والإشعارات، وجرّب الإلغاء لموعد أبعد من 24 ساعة.
  5. اكتب الملاحظات كما تظهر لك؛ سنحولها إلى حالات اختبار قابلة للتحقق.
